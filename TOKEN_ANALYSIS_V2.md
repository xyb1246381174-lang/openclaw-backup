## 🔍 背景分析详细步骤

### 1. 从合约找线索

**Basescan检查**：
```
□ Contract Creator（创建者地址）
□ TokenTracker（代币追踪）
□ Interactions（交互）
□ 创建时间（新旧）
```

**找创建者关联**：
```
□ 点击Creator地址
□ 看创建者创建的代币
□ 看创建者与其他项目的关联
```

### 2. 从DexScreener找信息

**关键信息**：
```
□ Token名称
□ Twitter链接
□ 描述/简介
□ 创建时间
```

**检查社交媒体**：
```
□ Twitter/X → 关注者/内容
□ Telegram → 成员/活跃度
□ 官网 → 团队介绍
```

### 3. 社交媒体深度分析

**Twitter分析**：
```
□ 账号关注者数量
□ 账号创建时间
□ 内容频率
□ 与谁互动
□ 有无名人关联
```

**关键问题**：
```
□ 创始人是谁？
□ 有无名人为其背书？
□ 是否有大V支持？
□ 背景是否可信？
```

### 4. 识别团队成员

**找团队线索**：
```
□ 官网团队页面
□ Twitter个人简介
□ LinkedIn
□ 过往项目经历
□ 有无公开身份
```

**背书分析**：
```
□ 名人/网红是否提及？
□ 有无VC/基金背书？
□ 是否有行业专家支持？
□ 关联项目有哪些？
```

## 🚨 背景分析常见陷阱

### 1. 只看数据不看人

```
错误：只看市值/流动性
正确：也要看谁在背后
```

### 2. 忽略社交媒体

```
错误：只看链上数据
正确：检查Twitter/Telegram
```

### 3. 不验证身份

```
错误：假设创始人可信
正确：验证公开身份
```

### 4. 不查关联

```
错误：只分析单个项目
正确：查看创建者其他项目
```
