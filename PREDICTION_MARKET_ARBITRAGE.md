## 一、什么是预测市场？

### 1.1 基本概念

```
预测市场 = Prediction Market

人们用真钱预测未来事件结果的平台

特点：
- 基于事件的概率交易
- 价格反映市场共识
- 事件结果揭晓后结算
```

### 1.2 主要平台

| 平台 | 链 | 特点 |
|------|-----|------|
| **Polymarket** | Polygon | 最大预测市场 |

## 二、预测市场价格机制

### 2.1 价格 = 概率

```
在预测市场中：
- Yes价格 = 该事件发生的概率
- No价格 = 该事件不发生的概率

例如：
- "Trump赢2024大选" → $0.65 = 65%概率
- "Trump输2024大选" → $0.35 = 35%概率

注意：$0.65 + $0.35 = $1.00
```

### 2.2 事件结算

```
事件结果揭晓后：
- 正确的一方 → 获得 $1.00
- 错误的一方 → 归零

例如：
- Trump赢了 → Yes代币值$1，No代币值$0
- Trump输了 → No代币值$1，Yes代币值$0
```

### 2.3 交易机制

```
买入Yes：
- 支付USDC
- 获得Yes代币
- 如果Yes赢了 → 获得$1
- 如果No赢了 → 获得$0

买入No：
- 支付USDC
- 获得No代币
- 如果No赢了 → 获得$1
- 如果Yes赢了 → 获得$0
```

## 三、预测市场套利原理

### 3.1 什么是套利？

```
套利 = 利用价格差异赚钱

在不同市场买入低价资产
同时卖出高价资产
赚取差价
```

### 3.2 预测市场套利类型

| 类型 | 描述 | 难度 |
|------|------|------|
| **跨平台套利** | Polymarket vs 其他市场 | ⭐⭐⭐ |
| **事件相关套利** | 相关事件的价格差异 | ⭐⭐⭐⭐ |
| **资金费率套利** | 利用资金费率差异 | ⭐⭐⭐ |
| **统计套利** | 利用统计规律 | ⭐⭐⭐⭐⭐ |

### 3.3 跨平台套利

```
核心原理：
在不同平台找到相同事件
利用价格差异套利

例如：
- Polymarket: "Trump赢" → $0.65
- 其他市场: "Trump赢" → $0.70

套利方法：
1. 在Polymarket买入$0.65的Yes
2. 在其他市场卖出$0.70的Yes
3. 赚取$0.05差价
```

### 3.4 事件相关套利

```
核心原理：
利用相关事件的价格关系
寻找价格偏离

例如：
事件A: "美联储3月降息" → $0.30
事件B: "美联储2024降息" → $0.60

如果事件A概率应该接近事件B的子集
价格偏离可能产生套利机会
```

## 四、Polymarket套利策略

### 4.1 基本套利策略

#### 策略1：Yes-No平衡套利

```
原理：Yes + No 应该 = $1.00

如果出现：
- Yes = $0.70
- No = $0.40
- 总和 = $1.10（不正常！）

套利方法：
1. 卖出高估的一方（No $0.40）
2. 买入低估的一方（Yes $0.70）
3. 等价格回归$1.00
4. 赚取$0.10差价
```

#### 策略2：跨平台价差套利

```
原理：同一事件，不同平台价格不同

例如：
- Polymarket Yes = $0.65
- Augur Yes = $0.70

套利方法：
1. 在Polymarket买入$0.65
2. 在Augur卖出$0.70
3. 赚取$0.05差价
```

#### 策略3：条件概率套利

```
原理：如果A发生则B一定发生
A的概率应该 >= B的概率

例如：
- "美联储3月降息" = $0.30
- "美联储2024降息" = $0.50

关系：3月降息 ⊂ 2024降息
所以：3月降息概率应该 <= 2024降息

如果出现相反情况 → 套利机会
```

### 4.2 进阶策略

#### 策略4：事件结果前买便宜一方

```
原理：事件结果揭晓前
便宜的一方可能价值被低估

方法：
1. 分析事件概率
2. 找到价格偏离基本面的市场
3. 买入被低估的一方
4. 等价格回归
```

#### 策略5：流动性套利

```
原理：大额订单会推动价格
可以利用大额订单的滑点

方法：
1. 发现大额订单
2. 预判价格移动方向
3. 提前建仓
4. 跟随趋势卖出
```

## 五、套利风险管理

### 5.1 风险类型

| 风险 | 描述 | 应对 |
|------|------|------|
| **平台风险** | 平台宕机/跑路 | 分散仓位 |
| **结算风险** | 事件结果争议 | 选择可信平台 |
| **流动性风险** | 无法及时成交 | 小仓位分批 |
| **执行风险** | 交易失败 | 设置滑点 |
| **法律风险** | 地区限制 | 了解法规 |

### 5.2 资金管理

```
建议：
- 单次套利 ≤ 10% 仓位
- 单日套利 ≤ 30% 仓位
- 保留50% 现金备用
- 设置止损线（-20%）
```

### 5.3 套利纪律

```
✅ 每次套利前计算利润
✅ 扣除Gas后还有利润才做
✅ 设置止盈止损
✅ 分散套利多个事件
✅ 不要追单
✅ 保留备用资金
```

## 七、套利案例分析

### 案例1：Yes-No价格偏离

```
事件：美国大选结果

市场状态：
- Yes价格: $0.75
- No价格: $0.35
- 总和: $1.10

分析：
总和应该是$1.00
$1.10偏离了$0.10

套利操作：
1. 卖出No（$0.35）
2. 买入Yes（$0.75）
3. 等价格回归$1.00
4. 利润: $0.10/代币
```

### 案例2：跨平台价差

```
事件：美联储利率决议

Polymarket:
- Yes: $0.60

Augur:
- Yes: $0.65

套利操作：
1. Polymarket买入Yes ($0.60)
2. Augur卖出Yes ($0.65)
3. 利润: $0.05/代币
```

### 案例3：条件概率套利

```
事件链：

事件A: "CPI < 3%" = $0.25
事件B: "通胀 < 3%" = $0.40

关系：A ⊂ B

分析：
如果CPI < 3%，则通胀必然 < 3%
所以事件A概率 ≤ 事件B概率

当前价格：
$0.25 ≤ $0.40 ✅ 合理
没有套利机会

如果：
事件A = $0.50
事件B = $0.40

$0.50 > $0.40 ❌ 不合理
套利机会：卖出A，买入B
```

## 十一、套利利润计算

### 简单套利计算

```
公式：
利润 = (卖出价 - 买入价) × 数量 - Gas费

例如：
买入100个Yes @ $0.60 = $60
卖出100个Yes @ $0.65 = $65
利润 = $5 - Gas费
```

### 考虑滑点

```
实际利润 = (实际卖出价 - 实际买入价) × 数量 - Gas费

注意：
- 大额订单有滑点
- 流动性差时滑点大
- 计算时要预留滑点
```
