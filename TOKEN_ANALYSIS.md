## 📈 第一部分：基本面分析

### 什么是基本面？

```
基本面 = 项目的内在价值
分析 = 这个币为什么有价值
```

### 核心指标

| 指标 | 含义 | 好 |
|------|------|---|
| **市值（Market Cap）** | 价格 × 流通量 | 低市值 = 机会大 |
| **流通量（Circulating Supply）** | 市场上流通的币 | 越多越好 |
| **总供应量（Total Supply）** | 总量 | 适中 |
| **最大供应量（Max Supply）** | 上限 | 有上限更好 |
| **FDV（完全稀释估值）** | 价格 × 总供应量 | 低FDV = 机会 |

### 代币经济学（Tokenomics）

#### 分配结构

```
理想分配：
├── 团队/顾问：15-20%
├── 社区/空投：20-30%
├── 流动性：20-30%
├── 生态基金：15-20%
└── 私募/公募：10-15%

危险信号 🚩：
├── 团队持币 > 50%
├── 私募解锁时间短
├── 巨鲸集中度高
└── 通胀模型差
```

#### 解锁时间表（Vesting）

```
好的 Vesting：
├── 线性解锁：2-4年
├── TGE解锁：<10%
└── 每月解锁：<5%

坏的 Vesting 🚩：
├── 立即解锁100%
├── 1-2年短期
└── 团队无锁仓
```

### 团队分析

| 检查项 | 好的信号 ✅ | 危险信号 🚩 |
|--------|------------|-------------|
| 团队身份 | 公开实名 | 匿名 |
| 经验 | 有Web3背景 | 无经验 |
| 历史 | 有成功项目 | 无记录 |
| 代码贡献 | 活跃开源 | 不透明 |

### 叙事分析

```
什么是叙事？
- 这个币的故事/主题
- 为什么存在
- 解决什么问题

土狗币叙事例子：
├── PEPE：青蛙表情包，社区文化
├── BONK：SOL链社区币
├── WIF：狗狗帽叙事
└── ACT：AI Agent概念

叙事热度 = 社区关注度
```

### 项目基本面清单

```
□ 1. 白皮书（做什么？）
□ 2. 代币经济学（怎么分？）
□ 3. 团队（谁在做？）
□ 4. 合作伙伴（谁支持？）
□ 5. 路线图（计划是什么？）
□ 6. 代码开源（透明吗？）
□ 7. 社区活跃度（Telegram/Discord）
```

## 📉 第二部分：技术分析

### K线基础

```
K线 = 蜡烛图
├── 绿色/上涨 = 价格上涨
├── 红色/下跌 = 价格下跌
└── 影线 = 最高/最低价
```

### 关键指标

| 指标 | 含义 | 用法 |
|------|------|------|
| **RSI** | 相对强弱指数 | <30超卖，>70超买 |
| **MACD** | 趋势指标 | 金叉买入，死叉卖出 |
| **MA** | 移动平均线 | 趋势方向 |
| **Bollinger Bands** | 波动区间 | 价格波动范围 |

### 常见形态

```
上涨信号 ✅：
├── 双底（W底）
├── 突破阻力
├── 放量上涨
└── 低点逐渐抬高

下跌信号 🚩：
├── 双顶（M顶）
├── 跌破支撑
├── 放量下跌
└── 高点逐渐降低
```

### 支撑与阻力

```
支撑 = 价格很难跌破的位置
阻力 = 价格很难突破的位置

用法：
- 支撑买入
- 阻力卖出
- 跌破支撑 = 止损
```

## ⛓️ 第三部分：链上分析

### 链上数据指标

| 指标 | 含义 | 好 |
|------|------|---|
| **Holder数量** | 持币地址数 | 增长 = 好 |
| **交易量** | 活跃度 | 放量 = 好 |
| **Dex流动性** | 深度 | >$10K = 好 |
| **Top Holder集中度** | 巨鲸分布 | <30% = 好 |
| **聪明钱流入** | 大户买入 | 流入 = 好 |

### 巨鲸追踪

```
巨鲸 = 大户/聪明钱

信号：
├── 巨鲸买入 = 可能涨
├── 巨鲸卖出 = 可能跌
├── 地址增加 = 社区壮大
└── 地址减少 = 社区萎缩

工具：
├── GMGN.ai
├── DexScreener
└── Bubblemaps
```

### 交易行为分析

| 行为 | 信号 |
|------|------|
| **聪明钱买入** | ✅ 看涨 |
| **做空被清算** | ✅ 看涨 |
| **做多被清算** | 🚩 看跌 |
| **大户地址增加** | ✅ 增长 |

### 安全检查

```
检查合约是否：
□ 1. 已审计
□ 2. 代码开源
□ 3. 没有后门
□ 4. 流动性已锁定
□ 5. 没有大额转账

工具：
├── Token Sniffer
├── Honeypot.is
└── RugDoc
```

```
基本面：
□ 1. 了解项目做什么

□ 3. 趋势方向
□ 4. 交易量
□ 5. 是否有突破形态

链上：
□ 1. Holder数量
□ 2. 巨鲸行为
□ 3. 流动性深度
□ 4. 安全检查
```

### 买入决策

```
必须满足：
✅ 叙事有吸引力
✅ 社区活跃
✅ 技术面有支撑
✅ 安全检查通过
✅ 只投亏得起的钱

可选：
├── KOL在推
├── 聪明钱买入
├── 突破形态
└── 低点逐渐抬高
```

### 卖出决策

```
止损线（必须设）：
├── 土狗：-50%必走
├── 主流：-30%走
└── 核心持仓：-50%走

止盈线（灵活）：
├── 土狗：2-5倍走
├── 强势：50%-100%走
└── 破位：走

不走的情况：
├── 还在叙事热度
├── 社区持续增长
├── 创新高
└── 核心逻辑没变
```

## 🎓 总结

### 代币分析三支柱

```
1. 基本面 = 为什么有价值
   ├── 代币经济学
   ├── 团队
   ├── 叙事
   └── 竞争力

2. 技术面 = 什么时候买/卖
   ├── K线形态
   ├── 支撑/阻力
   ├── 趋势
   └── 交易量

3. 链上 = 聪明钱在哪
   ├── 巨鲸行为
   ├── Holder变化
   ├── 流动性
   └── 安全检查
```

### 核心原则

```
1. DYOR（自己研究）
2. 不要FOMO（不要追高）
3. 设止损（保护本金）
4. 分散投资（不要全押一个）
5. 只投亏得起的钱

记住：
分析不能保证赚钱
但可以提高胜率
```
