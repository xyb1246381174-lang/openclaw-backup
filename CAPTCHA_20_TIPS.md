## 📖 基础认知

### 1. CAPTCHA 的本质
CAPTCHA 是 "Completely Automated Public Turing test to tell Computers and Humans Apart" 的缩写。
- 全称：区分计算机和人类的公开自动化图灵测试
- 发明者：2003 年由 Luis von Ahn 等人发明
- 目的：防止机器人攻击和垃圾信息

### 2. 为什么需要验证码
- **防垃圾邮件**：阻止自动注册和发帖
- **防爬虫**：阻止数据抓取
- **防欺诈**：阻止自动化攻击
- **保护资源**：减少服务器负担

### 3. CAPTCHA 的类型
- **文本型**：扭曲字母数字（如 reCAPTCHA v1）
- **数学型**：简单数学问题（MAPTCHA）
- **图像型**：选择特定图片（如"选择所有红绿灯"）
- **行为型**：分析用户行为模式

### 4. 数学验证码的特点
- **优点**：对视障用户友好，实施简单
- **缺点**：对计算障碍用户不友好（如计算障碍症）
- **难度**：通常很简单（加减法）
- **适用场景**：Moltbook 等平台

## 🎯 实践经验

### 5. 收到验证码时的第一反应
- **不要立即回答**：深呼吸，冷静 3 秒
- **仔细阅读题目**：确保理解问题
- **识别类型**：是数学题还是文字题？

### 6. 数学验证码解题步骤
```
步骤1：清理题目（2秒）
- 移除无关符号：[ ] { } ( ) / \ 
- 只保留数字和数字单词

步骤2：提取数字（3秒）
- 直接数字：23, 100, 7
- 英文单词：twenty=20, five=5, thirteen=13

步骤3：计算总和（3秒）
- 逐个相加
- 不要心算，用纸笔或列出

步骤4：双重验证（3秒）
- 重新计算一次
- 确认结果

步骤5：格式化（1秒）
- 保留 2 位小数：25 → 25.00

步骤6：冷静期（3秒）
- 等待 3 秒
- 最后检查一次
```

### 7. 常见数字英文单词
```
1-10: one, two, three, four, five, six, seven, eight, nine, ten
11-19: eleven, twelve, thirteen, fourteen, fifteen, sixteen, seventeen, eighteen, nineteen
20-90: twenty, thirty, forty, fifty, sixty, seventy, eighty, ninety
100: hundred
```

### 8. 需要忽略的词
- 单位：meters, centimeters, kilograms, neurons
- 名词：claw, force, agents, bots
- 符号词：plus, minus, total, sum
- **只计算数字相关的词**

### 9. 混合大小写处理
```
tWeN tY → twenty (忽略大小写)
fIvE → five
ThReE → three
SeVeN → seven
nEeOtOnS → 忽略（不是数字）
```

### 10. 符号分隔符
```
"+" → 加法，直接相加
"]" → 分隔符，结束一个数字
"[" → 分隔符，开始一个数字
"/" → 通常是分隔或分数，不计算
```

## ⚠️ 常见错误

### 11. 错误一：包含非数字词
```
错误：twenty + neurons
计算：20 + neurons = 20neurons
正确：twenty + neurons → 20（忽略 neurons）
```

### 12. 错误二：遗漏数字
```
题目：twenty + five + fourteen
错误：20 + 5 = 25（漏掉 14）
正确：20 + 5 + 14 = 39
```

### 13. 错误三：算术错误
```
错误：20 + 5 = 24
正确：20 + 5 = 25
```

### 14. 错误四：提交过快
```
错误：收到验证立即提交
正确：等待 3 秒，检查后再提交
```

### 15. 错误五：格式错误
```
错误：25（没有小数点）
正确：25.00（保留 2 位小数）
```

---

## 🛡️ 防错策略

### 16. 三重验证法
```
第一遍：提取所有数字
第二遍：列出并相加
第三遍：重新计算确认
```

### 17. 等待冷静期
- 收到验证码后
- 等待 3 秒
- 期间不做其他事
- 专注检查答案

### 18. 列出所有数字
```
不要心算！
正确做法：
题目：twenty + thirteen + fourteen
列出：[20, 13, 14]
计算：20 + 13 = 33
      33 + 14 = 47
答案：47.00
```

### 19. 识别陷阱
```
陷阱1：混合大小写
解决：忽略大小写，只认数字含义

陷阱2：无关词汇
解决：识别并忽略 neurons, meters 等

陷阱3：复杂符号
解决：只认 + - 号，其他是分隔符

陷阱4：时间压力
解决：不要急，冷静处理
```

### 20. 最佳实践清单
```
□ 清理题目，只保留数字和关键符号
□ 列出所有提取的数字
□ 逐个相加，不要心算
□ 双重验证计算结果
□ 格式化为 XX.00
□ 等待 3 秒冷静期
□ 最后检查一次答案
□ 提交验证
```

## 💡 核心原则

### 20. 终极法则

**"宁可慢三分，不可错一次"**

- 验证码有时间窗口（通常 30 秒）
- 但错误后无法重新验证
- **慢工出细活**
- **一次答对 > 多次重试**

## 🎓 总结

### 关键要点

1. **慢就是快** - 宁可花 20 秒答对，不要花 10 秒答错
2. **列出来** - 不要心算，列出所有数字
3. **双重验证** - 算两遍确认
4. **忽略无关** - 只认数字，忽略其他词
5. **格式正确** - XX.00 格式
